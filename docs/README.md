## 文档命名规范

### 前缀定义

| 前缀 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `PRD-` | Product Requirements Document | 产品级需求文档（跨模块） | `PRD-总体需求.md` |
| `REQ-` | Requirements | 模块级需求文档 | `REQ-M1-串口通信管理.md` |
| `TSD-` | Technical Selection Document | 技术选型文档 | `TSD-M1-技术选型.md` |
| `DSD-` | Detailed Design Document | 详细设计文档 | `DSD-M1-详细设计.md` |
| `IMP-` | Implementation Plan | 实施计划文档 | `IMP-M1-实施计划.md` |
| `RPT-` | Report | 实施报告文档 | `RPT-M1-实施报告.md` |
| `TC-` | Test Case | 测试用例文档 | `TC-M1-测试用例.md` |
| `REV-` | Review | 审查报告文档 | `REV-M1-审查报告.md` |

### 命名规则

```
{前缀}-{模块编号}-{描述}.md
```

- 模块编号：`M1` ~ `M6`，跨模块文档不加编号
- 描述：简洁中文描述，用连字符分隔
- 全部使用 `.md` 格式
- 示例：`REQ-M1-串口通信管理.md`、`DSD-M3-压测执行流程.md`、`TC-M1-URC处理测试.md`

### 版本管理

- 每份文档头部包含版本号和日期
- 文档通过 git 版本控制管理，不创建历史副本
- 文档变更通过 git 提交记录追溯

### 实现状态

M1-M7 七个模块已完成第一阶段实现（Python 3.11+ / pyserial / PySide6 / Typer / Pydantic / Jinja2）：

- **领域层（M2/M4）**：用例模型、YAML 解析、模板替换器、表达式求值器、断言求值、extract、报告数据模型与聚合器 — 纯函数，无 I/O
- **基础层（M1）**：串口接口（Protocol）、pyserial 连接、后台读线程、响应完整性判定、URC 分流、数据流分块发送、HEX+TEXT 原始日志、多端口管理、热插拔重连、FakeSerial
- **引擎层（M3）**：串行调度器、单步执行（retry/poll/when/extract/assert/on_failure）、压测循环、进度事件
- **环境配置（M7）**：env.yaml 加载、`{{group.param}}` 点号引用、模板上下文注入
- **报告（M4）**：控制台实时输出 + 纯静态 HTML（内联 CSS、`<details>`、无 JS）
- **CLI（M5）**：`run`/`list`/`gui` 子命令、`--version`/`-V` 版本标志、`--port` 复合表达式、标签过滤、dry-run、`--vsim` 虚拟模组模式、`--baud` 覆盖
- **GUI（M6）**：PySide6 两栏布局（侧栏导航 + MDI 选项卡外壳）、选项卡类型注册表、6 类选项卡（手动调试/用例执行/实时监控/执行进度/报告查看/环境配置）、浅深主题切换。详细实现现状见 REQ-M6 §14

质量门禁：mypy strict 全绿、ruff 全绿、pytest 150+ 项通过。
