### 5.1 AT+CSMS — 选择短信服务

用于支持的短消息包括：发送（SMS-MO）、接收（SMS-MT）、小区广播（SMS-CB）。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>+CSMS: <service>,<mt>,<mo>,<bm> <CR><LF>OK<CR><LF>` |
| 执行 | `AT+CSMS=<service><CR> Or` | `<CR><LF>ERROR<CR><LF> <CR><LF>+CSMS: <service>,<mt>,<mo>,<bm>` |
| 查询 | `AT+CSMS?<CR>` | `<CR><LF>OK<CR><LF> <CR><LF>+CSMS: (list of supported <service>s)` |
| 测试 | `AT+CSMS=?<CR>` | `<CR><LF>OK<CR><LF>` |

**参数**

默认值为0,1,1,1。

| <service> | 0：GSM03.40 and GSM03.41；SMS 相关AT 指令支持 GSM07.05 Phase 2 |
| --- | --- |
|  | 1：GSM03.40 and GSM03.41；SMS 相关AT 指令支持 GSM07.05 Phase 2+ |
| <mt>,<mo>,<bm> | 0：不支持 |
|  | 1：支持 |

**示例**

```
AT+CSMS=1 设置选择短信服务为1
+CSMS: 1,1,1,1
OK
AT+CSMS? 查询指令当前设置的参数值
+CSMS: 1,1,1,1
OK
AT+CSMS=?
+CSMS: (0-1)
OK
```


### 5.2 AT+CPMS — 首选短信存储器

### 5.4 AT+CSCS — 设置TE 字符集

设置TE 字符集格式。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>OK<CR><LF>` |
| 设置 | `AT+CSCS[=<chest>]<CR> Or` | `<CR><LF>ERROR<CR><LF> <CR><LF>+CSCS: <chset>` |
| 查询 | `AT+CSCS?<CR>` | `<CR><LF>OK<CR><LF> <CR><LF>+CSCS: (list of supported <chset>s)` |
| 测试 | `AT+CSCS=?<CR>` | `<CR><LF>OK<CR><LF>` |

**参数**

默认字符集为"PCCP936"。

| <chset> | "GSM"：GSM 默认字母表（GSM03.38.6.2.1） |
| --- | --- |
|  | "IRA"：国际参考字母表(international reference alphabet)(ITU-T T.50) |
|  | "UCS2" ： 16-bit universal multiple-octet coded character set |
|  | (USO/IEC10646)。UCS2 字符串被转换成一个十六进制数（0x0000～ |
|  | 0xFFFF），只有在相应语句中的字符串才用UCS2 编码，其余的命令和响 |
|  | 应仍旧是IRA 字母表格式的。 |
|  | "PCCP936"：等同GBK 编码格式 |
|  | "IRA"：国际参考字母表(international reference alphabet)(ITU-T T.50) |

**示例**

```
AT+CSCS="GSM" 设置“IRA”字符集
OK
AT+CSCS? 查询当前字符集格式
+CSCS: "PCCP936"
OK
AT+CSCS=? 查询模组支持的字符集格式， 返回字符集格式的列表
+CSCS: ("GSM","HEX","PCCP936","UCS2",”IRA”)
OK
```


### 5.5 AT+CNMI — 设置短信指示格式

设置模组从网络接收到短信息后如何通知用户。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  | `AT+CNMI=[<mode>[,<mt>[` | `<CR><LF>OK<CR><LF>` |
| 设置 | `,<bm>[,<ds>[,<bfr>]]]]]<CR Or >` | `<CR><LF>ERROR<CR><LF> <CR><LF>+CNMI: [<mode>[,<mt>[,<bm>[,<ds>[,<bfr>]]]]]` |
| 查询 | `AT+CNMI?<CR>` | `<CR><LF>OK<CR><LF> <CR><LF>+CNMI: (list of supported <mode>s),(list of supported <mt>s),(list of supported <bm>s),(list of supported` |
| 测试 | `AT+CNMI=?<CR>` | `<ds>s),(list of supported <bfr>s) <CR><LF>OK<CR><LF>` |

**参数**

默认值为0, 0, 0, 0, 0；  建议设置为+CNMI: 2,1,0,0,0（新短信内容存贮SIM 卡而不直接显示）或+CNMI: 2,2,0,0,0（新短信  内容直接显示而不存贮在SIM 卡）； 短信根据存储位置分为class0~3 四类  Class 0：只显示，不储存  Class 1：储存在ME 内存中  Class 2：储存在SIM 卡中  Class 3：直接传输到TE

| <mode> | 设置收到短信后向用户发送的指示方式。 |
| --- | --- |
|  | 0：短信指示代码在模组中缓存,如果TA 已满，代码可存贮在其它地方或者将最旧的代码 |
|  | 丢弃，并用最新接收到的代码去替代。 |
|  | 1：当On-line State 时，舍弃保存的短信指示代码并拒绝新的指示代码，其它情况下，直 |
|  | 接将代码显示在终端设备上。 |
|  | 2：当模组在On-line State 时，短信指示代码在模组中缓存，当连接释放后，将短信指示 |
|  | 代码通过串口输出，在其他state 下，直接将代码显示在终端设备上。 |
|  | 3：当模组在On-line State 时，短信指示代码和其它数据一起传输，代码显示在终端设备 |
|  | 上。 |
| <mt> | 设置新短信指示代码的格式，默认值为0 |
|  | 0：不向终端发送新短信指示代码。 |
|  | 1：新短信指示代码格式为+CMTI: "MT",<index>，短信内容存贮而不直接显示。 |
|  | 2：新短信指示代码格式为+CMT: <oa>,<scts>,<tooa>,<lang>,<encod>,<priority> |
|  | [,<cbn>],<length><CR><LF><data>(文本模式)，短信内容直接显示而不存贮。 |
|  | 3：使用<mt>=2 所定义的主动上报代码直接将短信指示代码传送到终端设备，其它模式 |
| --- | --- |
|  | 的信息指示代码和<mt>=1 相同。 |
| <bm> | 设置新小区广播指示代码的格式，默认值为1。 |
|  | 0：不发送新小区广播的指示信息，小区广播也不会存贮 |
|  | 2：新小区广播指示代码格式为+CBM: <oa>,[<alpha>,]<scts>[,<tooa>,<length>] |
|  | <CR><LF><data>(文本模式),小区广播内容直接显示而不存贮。 |
| <ds> | 短信发送的状态报告，默认值为1。 |
|  | 0：无短信发送状态报告 |
|  | 1：短信发送状态报告代码格式为+CDS: <fo>,<mr>,[<ra>],[<tora>],<scts>, |
|  | <dt>,<st>(文本模式) |
| <bfr> | 默认值为 0。 |
|  | 0：当<mode>设为1、2 时，存贮在TA 中的该指令定义的代码将会被发送到TE，模组在 |
|  | 传送代码之前，会先返回"OK"。 |
|  | 1：当<mode>设为1、2 时，存贮在TA 中的该指令定义的代码将会被清除。 |

**示例**

```
AT+CNMI=1,1,0,0,0 设置收到短信后向用户发送的指示方式，新短信指示代码的格式,新小区
OK 广播指示代码的格式，短信发送的状态报告
AT+CNMI=? 查询指令参数可设置的范围
+CNMI: (0-3),(0-3),(0-2),(0-1),(0-1)
OK
AT+CNMI? 查询指令当前设置参数值
+CNMI: 1, 1, 0, 0, 0
OK
```

| --- | --- | --- |
