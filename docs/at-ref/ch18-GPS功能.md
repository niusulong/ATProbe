### 18.1 AT$MYGPSPWR — GPS 指令开关

GPS 开关指令。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>OK<CR><LF>` |
| 设置 | `AT$MYGPSPWR=<n><CR>` | `Or <CR><LF>ERROR<CR><LF>` |

**参数**

开启GPS 服务会占用模组较多资源，如果不需要GPS 服务，可以关闭掉GPS 服务。  请求成功后，模组从开启GPS 到GPS 定位需要一段时间，具体时间和环境因素有关，请注意天线  匹配。

| <n> | 请求选择 |
| --- | --- |
|  | 0：关闭GPS 服务 |
|  | 1：开启GPS 服务 |

**示例**

```
AT$MYGPSPWR=1 开启GPS 服务。
OK
AT$MYGPSPWR=0 关闭GPS 服务。
OK
```

### 18.4 AT$MYGNSSSEL — 选择定位系统

选择辅助定位系统。 使用时需要先开启GPS 定位系统，然后选择需要的辅助定位系统；  GPS 定位信息通过$MYGPSPOS 查询，辅助系统的定位信息通过$MYGNSSMSG 查询；  V003 及之后的版本支持该指令。 

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>OK<CR><LF>` |
| 执行 | `AT$MYGNSSSEL=<n><CR>` | `Or <CR><LF>ERROR<CR><LF>` |

**参数**

| <n> | 辅助定位系统选择 |
| --- | --- |
|  | 0：BD |
|  | 1：GLONASS |
|  | 3：BD、GLONASS |

**示例**

```
AT$MYGNSSSEL=1 OK 打开GLONASS 辅助定位。
AT$MYGNSSSEL=0 OK 打开北斗辅助定位。
```

### 18.6 AT+SETSERVER — AGPS 设置服务器

设置AGPS 的服务器。 SETSERVER 必须在GPS 打开之前设置。  AGPS 服务器需要运营商部署或第三方提供，示例中supl.qxwz.com 仅用于测试。  专网卡、物联网卡请确认是否能访问设置的URL。 

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  | `AT+SETSERVER=<n>,<url>,<port>,[<` | `<CR><LF>OK<CR><LF>` |
| 设置 | `usr>,<pwd>,<pt>]<CR>` | `Or <CR><LF>ERROR<CR><LF>` |

**参数**

| <n> | 当前制式类型。 |
| --- | --- |
|  | 1:3GPP2 |
|  | 3:3GPP (目前仅支持GSM、LTE 制式下使用) |
| <url> | 服务器地址，支持IPv4、IPv6、URL。 |
| <port> | 服务器端口。 |
| <usr> | 登录服务器用户名 |
| <pwd> | 登录密码 |
| <pt> | 数据服务提供商(provider type) |
|  | 0:泰斗 |
|  | 1:中科微 |
|  | 2:和芯星通 |

**示例**

```
AT+SETSERVER=3,agnss.neoway.com,80,,,2 设置AGPS 的服务器地址。
OK
```


### 18.7 AT$MYGPSCFG — GPS 设置

设置GPS 相关参数。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>OK<CR><LF>` |
| 设置 | `AT$MYGPSCFG=<type>,<value><CR>` | `Or <CR><LF>ERROR<CR><LF> <CR><LF>$MYGPSCFG:` |
| 查询 | `AT$MYGPSCFG?<CR>` | `<type>,<value><CR><LF> <CR><LF>OK<CR><LF>` |

**参数**

| <type> | 设置类型。 |
| --- | --- |
|  | 0：设置GPS 更新频率 |
|  | 1：设置NMEA 数据上报 |
|  | 2：设置上报的NMEA 格式 |
| 其他：预留 |
| --- |
| 根据<type>配置对应的值。 |
| type =0 时，表示更新间隔，取值范围1000~60000ms |
| type=1 时，value 为0 关闭上报，1 打开上报，默认打开 |
| type=2 时，value 为十进制数值，下面是对应NMEA 格式的mask，默认全打开 |
| GPGGA 0x00000001 |
| GPRMC 0x00000002 |
| GPGSV 0x00000004 |
| GPGSA 0x00000008 |
| GPVTG 0x00000010 |
| GLGSV 0x00000080 |
| GNGSA 0x00000100 |
| GNGGA 0x00000200 |
| GNGLL 0x00000400 |
| BDGSV 0x00000800 |
| GNRMC 0x00001000 |
| GNVTG 0x00002000 |
| GNZDA 0x00004000 |
| GPZDA 0x00008000 |
| GPTXT 0x00010000 |
| GPGLL 0x00020000 |

**示例**

```
AT$MYGPSCFG=0,1000 设置GPS 更新频率为1s，NMEA 口每隔1s 上报一次数据。
OK
AT$MYGPSCFG? 查询参数设置。
$MYGPSCFG: 0,1000
$MYGPSCFG: 1,1
$MYGPSCFG: 2,4294967295
OK
AT$MYGPSCFG=2,1 设置仅上报GPGGA 数据。
OK
AT$MYGPSCFG=1,0 关闭NMEA 口上报。
OK
```
