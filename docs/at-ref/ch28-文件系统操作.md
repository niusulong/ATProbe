### 28.1 AT+FSWF — 写文件

写文件指令。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  | `AT+FSWF=<file_name>,<mode>,<size>` | `<CR><LF>OK<CR><LF> Or` |
| 执行 | `,<time><CR>` | `<CR><LF>ERROR<CR><LF> Or <CR><LF>+FSWF: Timeout!<CR><LF>` |

**参数**

写入文件的数据，不允许超过<size>值；  用户盘的大小为1M，即能存储文件的总大小为1M；  若支持SD 卡，文件名带“/sdcard0” (区分大小写)可操作SD 卡文件。  若支持外挂flash ，文件名带”/dataflash”( 区分大小写) 可操作外挂flash 文件, 需要使用  +NWYSPIFLASH 进行初始化。

| <file name> _ |  | 文件名，支持路径。文件名或路径不存在则创建文件名或路径。 |
| --- | --- | --- |
|  |  | 文件路径是相对于用户目录而言的。 |
|  |  | 长度不超过120 个字符。 |
|  | <mode> | 模式 |
|  |  | 0：如果文件已经存在，从文件开始位置写入，会覆盖原数据 |
|  |  | 1：如果文件已经存在，从文件末尾位置写入 |
| <size> | <size> | 数据大小，取值范围0~1024*1024，单位：字节 |
|  |  | 写入文件的数据，不允许超过<size>值。 |
|  | <time> | 超时时间，取值范围0~240000，单位：ms |

**示例**

```
AT+FSWF="test.txt",1,1024,10000 写文件成功，test.txt 文件写入1024 字节数据，数据如： start012…。
>
OK
AT+FSWF="test.txt",1,1024,10000 写文件超时，串口输入数据大小小于1024，时间超过10s，返回超时提 示。
>
+FSWF: Timeout!
AT+FSWF="test.txt",1,1024,60001 参数超出可设置范围，返回ERROR。
ERROR
AT+FSWF=”/sdcard0/test.txt”,1,10,60000 往SD 卡中test.txt 文件写入数据。
>
OK
AT+FSWF=/dataflash/test.txt,1,10,120000 往外挂flash 中10.txt 文件写入数据。
>
OK
AT+FSWF="/dir/test.txt",1,1024,10000 往/dir 目录中的test.txt 文件写入1024 字节数据
>
OK
```


### 28.2 AT+FSRF — 读文件

读文件指令。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  | `AT+FSRF=<file_name>,<mode>,<size>` | `<CR><LF>+FSRF: <size>,<content> <CR><LF>OK<CR><LF>` |
| 执行 | `[,<position>]<CR>` | `Or <CR><LF>ERROR<CR><LF>` |

**参数**

读取文件的数据大小，不允许超过文件总大小；  <size>与<position>由文件大小决定；  若支持SD 卡，文件名带“/sdcard0” (区分大小写)可操作SD 卡文件。  若支持外挂flash ，文件名带”/dataflash”( 区分大小写) 可操作外挂flash 文件, 需要使用  +NWYSPIFLASH 进行初始化。

| <file name> _ |  | 文件名，支持路径。文件名或路径不存在则上报错误。 |
| --- | --- | --- |
|  |  | 文件路径是相对于用户目录而言的。 |
|  |  | 长度不超过120 个字符 |
| <mode> |  | 0：从文件开始位置读数据 |
|  |  | 1：从文件<position>位置读数据 |
|  | <size> | 数据大小，不允许超过所读文件的大小，可以为0 |
|  | <position> | 读文件的开始位置，当<mode>=1 时有效，<size>与<position>由文件大小决定。 |
|  | <content> | 读取的文件内容。 |

**示例**

```
AT+FSRF="test.txt",0,10 从test.txt 文件的开始位置读10 个字节数据。
+FSRF: 10,start01234
OK
AT+FSRF="test.txt",0,0 从test.txt 文件的开始位置读0 个字节数据。
+FSRF: 0,
OK
AT+FSRF="test.txt",0,1025 <size>超过文件大小，返回ERROR。
ERROR
AT+FSRF="test.txt",1,20,2 从test.txt 文件的第二个字节的位置开始，读20 个字节的数据，读取成 功。
+FSRF: 20,tart0123456789012345
OK
AT+FSRF="test.txt",1,0,2 从test.txt 文件的第二个字节的位置开始，读0 个字节的数据。
+FSRF: 0,
OK
AT+FSRF="test.txt",1,10,1025 <position>参数超出范围（position 大于文件大小），返回ERROR。
ERROR
AT+FSRF="/sdcard0/test.txt",0,10 读取SD 卡test.txt 文件内容。
+FSRF: 10,start01234
OK
AT+FSRF=/dataflash/test.txt,0,10 读取外挂flash 中test.txt 文件内容。
+FSRF: 10,aaaaaaaaaa
OK
```


### 28.3 AT+FSRFEX — 读文件拓展指令

读文件拓展指令。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  | `AT+FSRFEX=<file_name>,<checksum_` | `<CR><LF>+FSRFEX: <lenth>,<checksum>,<content>` |
| 执行 | `mode>,<mode>,<size>[,<position>]` | `<CR><LF>OK<CR><LF> Or <CR><LF>ERROR<CR><LF>` |

**参数**

读取文件的数据大小，不允许超过文件总大小；  <size>与<position>由文件大小决定；  若支持SD 卡，文件名带“/sdcard0” (区分大小写)可操作SD 卡文件。  若支持外挂flash ，文件名带”/dataflash”( 区分大小写) 可操作外挂flash 文件, 需要使用  +NWYSPIFLASH 进行初始化

| <file name> _ |  | 文件名，长度不超过50 个字符。 |
| --- | --- | --- |
|  |  | 文件名，支持路径，文件名或路径不存在则上报错误。 |
|  |  | 文件路径是相对于用户目录而言的。 |
|  |  | 长度不超过120 个字符。 |
| <checksum mode> _ |  | 校验类型 |
|  |  | 1：异或校验法 |
|  |  | 2~99: 预留 |
| <mode> |  | 模式 |
|  |  | 0：从文件开始位置读数据 |
|  |  | 1：从文件<position>位置读数据 |
|  | <size> | 数据大小，不允许超过所读文件的大小，不为0。 |
| <position> | <position> | 读文件的开始位置，当<mode>=1 时有效，不为 0，目前最大支持到 |
|  |  | 0x40000000。 |
|  | <lenth> | 所读文件的大小。 |
|  | <content> | 所读文件的内容。 |
|  | <checksum> | 校验码，校验方式由<checksum mode>指定 _ |

**示例**

```
AT+FSRFEX="test.txt",1,0,10 从test.txt 文件的开始位置读10 个字节数据。
+FSRFEX: 10,XXXX,start01234
OK
AT+FSRFEX=/sdcard0/123.txt,1,0,10 从SD 卡中的123.txt 文件开始位置读10 个字节数据。
+FSRFEX: 10,XXXX,1234567890
OK
AT+FSRFEX=/dataflash/test.txt,1,0,10 从外挂flash 中的test.txt 文件开始位置读10 个字节数据。
+FSRFEX: 10,6425,aaaaaaaaaa
OK
AT+FSRFEX="/dir/test.txt",1,0,10 读取/dir 目录下的test.txt 文件。
+FSRFEX: 10,XXXX,start01234
OK
```

### 28.7 AT+FSLS — 获取用户盘剩余空间大小

获取用户盘剩余空间大小指令。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>+FSLS:<size>` |
| 执行 | `AT+FSLS=<directory><CR>` | `<CR><LF>OK<CR><LF> <CR><LF>+FSLS: <size> <CR><LF>OK<CR><LF>` |
| 查询 | `AT+FSLS?<CR>` | `Or <CR><LF>+FSLS: DiskInfo Not Right!<CR><LF>` |

**参数**

文件系统的块大小为500 字节，以实际文件占用块大小为准，SD 卡或者外挂flash 不同型号块大小  可能会不同。 若支持SD 卡，文件名带“/sdcard0” (区分大小写)可操作SD 卡文件。  若支持外挂flash ，文件名带”/dataflash”( 区分大小写) 可操作外挂flash 文件, 需要使用  +NWYSPIFLASH 进行初始化。

| <directory> | 查询SD 卡或者外挂flash 剩余空间时需要指定路径。 |
| --- | --- |
| <size> | 用户盘剩余空间大小。 |

**示例**

```
AT+FSLS? 用户盘剩余空间大小为64500 字节， 64500=500*129，即剩余129 个数据块（500 字节/块）。
+FSLS: 64500
OK
AT+FSLS=/sdcard0 查看SD 卡剩余大小。
+FSLS: 1535345345
OK
AT+FSLS=/dataflash 查看外挂flash 剩余空间大小。
+FSLS: 15878544
OK
```

### 28.9 AT+FSRN — 重命名文件

重命名文件指令。 重命名文件后的文件若存在，则会覆盖原来的文件。 若需要修改的文件不存在，则会返回错误。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>OK<CR><LF>` |
| 执行 | `AT+FSRN=<src_file_name>,<dst_file_name><CR>` | `Or <CR><LF>ERROR<CR><LF>` |

**参数**

若支持SD 卡，文件名带“/sdcard0” (区分大小写)可操作SD 卡文件。  若支持外挂flash ，文件名带”/dataflash”( 区分大小写) 可操作外挂flash 文件, 需要使用  +NWYSPIFLASH 进行初始化。

| <src file name _ _ > <dst file name _ _ > | 需要修改的文件名，支持路径，长 |
| --- | --- |
|  | 度不超过120 个字符。 |
|  | 重命名之后的文件名，支持路径， |
|  | 长度不超过120 个字符。 |
| <src file name _ _ |
| --- |
| > |
| < |
| > |

**示例**

```
AT+FSRN="test.txt","dst.txt" 重命名文件成功。
OK
AT+FSRN="test1.txt","dst.txt" 重命名文件失败。
ERROR
AT+FSRN=/dataflash/test.txt,/dataflash/123.txt 重名名外挂flash 下面的文件。
OK
AT+FSRN="/dir/test.txt","/dir/dst.txt" 重命名文件成功。
OK
```


### 28.10 AT+NWSPIREAD — SPI 读取文件

SPI 读文件指令。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  | `AT+NWSPIREAD=<spi_id>,<file_name>,<mode>,` | `<CR><LF>OK<CR><LF>` |
| 执行 | `<size>[,<position>]<CR>` | `Or <CR><LF>ERROR<CR><LF>` |

**参数**

读取文件的数据大小，不允许超过文件总大小；  <size>与<position>由文件大小决定；  若支持SD 卡，文件名带“/sdcard0” (区分大小写)可操作SD 卡文件。  若支持外挂flash ，文件名带”/dataflash”( 区分大小写) 可操作外挂flash 文件, 需要使用  +NWYSPIFLASH 进行初始化。

| <spi id> _ |  | SPI 号，整数类型 |
| --- | --- | --- |
|  |  | 1：SPI1 |
|  |  | 2：SPI2 |
|  | <file name> _ | 文件名，长度不超过50 个字符 |
| <mode> | <mode> | 0：从文件开始位置读数据 |
|  |  | 1：从文件<position>位置读数据 |
|  | <size> | 数据大小，不允许超过所读文件的大小，可以为0 |
| <position> | <position> | 读文件的开始位置，当<mode>=1 时有效，<size>与<position>由文件大小决定。 |

**示例**

```
AT+NWSPIREAD=1,"test",0,200 从test.txt 文件的开始位置读200 个字节数据。
OK
```
