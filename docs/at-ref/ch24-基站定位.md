### 24.1 AT+CIPGSMLOC — 获取基站定位信息

获取基站定位信息。 获取的是GPS 坐标。 当前经纬度是有效的，精度是保留项，默认为0.0。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>+CIPGSMLOC: <fail_string><CR><LF> Or <CR><LF>OK<CR><LF> <CR><LF>+CIPGSMLOC: {<result_string>} <CR><LF>+CIPGSMLOC: OK<CR><LF> Or` |
| 执行 | `AT+CIPGSMLOC[=<n>]<CR>` | `<CR><LF>OK<CR><LF> <CR><LF><code> <CR><LF>+CIPGSMLOC: FAIL<CR><LF> Or <CR><LF>OK<CR><LF> <CR><LF>+CIPGSMLOC: TIMEOUT<CR><LF>` |

**参数**

| <n> | 请求选择。 |
| --- | --- |
|  | 0：主动关闭定位请求 |
|  | 1：选择多基站定位请求 |
| <fail string> _ | 请求失败的提示字符。 |
|  | CONTACT FAIL |
|  | LINK FAIL |
|  | LINK NOT FREE |
|  | ERROR |
|  | 包含经纬度的字符串。 |
| <code> | 请求发送成功，但无经纬度返回时，服务器的响应码。 |
| --- | --- |
|  | 401：没有权限访问 |
|  | 400：请求在解析过程中出错 |
|  | 404：请求合法，但是所查基站未被收录，因此无法计算出结果 |
|  | 408：服务器解析超时 |
|  | 500：服务器内部错误 |

**示例**

```
AT+CIPGSMLOC 指令成功。 模组位置信息上报。
OK
+CIPGSMLOC:{"location":{"lat":22.69083,"lng":113.985228},
"accuracy":0.0}
+CIPGSMLOC: OK
AT+CIPGSMLOC 服务器域名解析失败。
+CIPGSMLOC: CONTACT FAIL
AT+CIPGSMLOC 服务器链接失败。
+CIPGSMLOC: LINK FAIL
AT+CIPGSMLOC 位置请求成功。 但所查基站未被收录，无法计算出结果。
OK
+CIPGSMLOC: 404
+CIPGSMLOC: FAIL
AT+CIPGSMLOC=1 请求多基站定位。 上一次请求还未响应，链路还未释放，再次请 求，提示链路处于占用状态。
OK
AT+CIPGSMLOC=1
+CIPGSMLOC: LINK NOT FREE
AT+CIPGSMLOC=0 主动关闭请求，此时链路会得到释放。
OK

OK

"lng": 113.98586121790129},"accuracy": 0.0}

```
