### 31.8 AT$MYBAND — 锁定GSM 频段

锁定指定频段。 此锁频段指令只适用于GSM900/1800 双频模组。该命令用于锁定模块注册到GMS 900MHz 或 GSM 1800MHz 频段的蜂窝小区，并使用参数0 来解除锁定的GSM band，恢复制式为Auto(LTE 优先） 模式，在这个情况下模块可以在所有支持的band 来尝试注册。 注意：使用AT$MYBAND=0，解锁模块锁定的GSM band 后，此时模块的制式就会变为Auto （lte+gsm， lte 优先），客户若需要设置锁定2G 所有的频段，使用锁定制式指令（AT$MYSYSINFO）， 将制式锁定为GSM only 即可，AT$MYBAND=1 或2 用于设置特定的单一GSM 频段。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  |  | `<CR><LF>OK<CR><LF>` |
| 执行 | `AT$MYBAND=<band><CR>` | `Or <CR><LF>ERROR<CR><LF> <CR><LF>$MYBAND: <band>` |
| 查询 | `AT$MYBAND?<CR>` | `<CR><LF>OK<CR><LF>` |

**参数**

| <band> | 锁定的频段，此参数设定掉电保存。 |
| --- | --- |
|  | 0：自动选择频段:解锁当前锁定的GSM band，模块恢复为auto 制式（lte 优先），后自 |
|  | 动选择一个频段的小区来接入网络，默认值就为0 |
|  | 1：GSM EGSM 900 _ _ |
|  | 2：GSM DCS 1800 _ _ |

**示例**

```
AT$MYBAND=1
OK
AT$MYBAND?
$MYBAND: 1
OK
```

### 31.11 AT$MYSYSINFO — 当前网络运行制式的查询、锁定

当前网络运行制式的查询、锁定。 当通过该指令将网络制式设置为非AUTO 模式，模组会开启自动检测网络状态功能，即实时监测模 组是否能够注册上网络。 若检测到模组注册上网络，则模组会判断注册的网络是否与要求相符，如果不相符，则去切换为所 设置的网络。 若检测到模组掉网，且在所设定的时间（通过$MYNETAUTO 设置）内未能再次找回网络，模组会 自动切换到AUTO 模式。

**命令格式**

| 类型 | 命令 | 响应格式 |
| --- | --- | --- |
|  | `AT$MYSYSINFO=<Set_SysMo` | `<Set_SysMode> <CR><LF>OK<CR><LF> <CR><LF>OK<CR><LF>` |
| 设置 执行/查询 | `de><CR> AT$MYSYSINFO<CR>` | `Or <CR><LF>ERROR<CR><LF> <CR><LF>$MYSYSINFO: <CurrentMode >,<mnc> <CR><LF>OK<CR><LF>` |
| 测试 | `AT$MYSYSINFO=?<CR>` | `<CR><LF>$MYSYSINFO: value range of` |

**参数**

| <Set SysMode> _ | 网络制式 |
| --- | --- |
|  | 1：AUTO |
|  | 2：2G(包括GSM、EDGE) |
|  | 4：4G(包括FDD-LTE、TDD-LTE) |
| <Current SysMode> _ | 网络制式 |
|  | 0：No service |
|  | 2：2G(包括GSM、EDGE) |
|  | 4：4G(包括FDD-LTE、TDD-LTE) |
| <mnc> | 网络运营商代码 |
|  | 00：注册失败 |
|  | 01：中国移动 |
|  | 02：中国联通 |
|  | 03：中国电信 |
|  | 04：未知 |

**示例**

```
AT$MYSYSINFO 当前注册在中国移动的4G 网络下。
$MYSYSINFO: 4,01
OK
AT$MYSYSINFO=1 设置网络模式为自动。
```

### 31.27 AT+NWATDFOTA — 本地OTA 升级
