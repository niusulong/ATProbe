## 2. M1 串口通信管理 — 已确认

文档已定稿，关键设计决策回顾：

- **参数分类**：分为连接级（硬件参数，连接期间不变）和行为级（超时/间隔，支持三级优先：步骤>用例>全局）
- **热插拔**：自动重连，固定间隔3秒+最大重试10次，重连期间暂停当前用例
- **多串口**：最多4个并行，独立执行+跨端口联动（变量共享、URC跨端口触发、单用例跨端口执行）
- **URC**：四种匹配模式（前缀/正则/包含/精确），三种触发动作（记录日志/触发测试步骤/执行命令），等待响应期间也提取URC
- **日志**：HEX+TEXT双格式同文件，按用例粒度，不自动清理
- **异常**：记录失败+继续执行下一个用例

## 3. M2 测试用例定义 — 草稿待审阅

文档已生成，用户已审阅部分内容并提出修改意见。关键设计决策：

### 3.1 已确认的设计决策

| 讨论要点 | 决策 |
|---------|------|
| YAML结构 | 统一结构，三种场景共用一套schema |
| 断言类型 | 四种：contains / matches / equals / not_contains |
| 变量提取 | 正则提取（捕获分组） |
| 流程控制 | 条件跳过（when字段），支持比较运算符和&&/||组合 |
| 套件组织 | 套件文件引用用例文件（不内嵌） |
| 标签系统 | 纯自定义标签，无预定义 |
| 前置条件 | 命令检查式，失败跳过整个用例 |
| 变量传递 | 两级作用域：用例级（默认）和跨端口级（通过 export 导出）。用例间不传递，跨端口联动场景通过 export 机制共享 |

### 3.2 用户反馈及修改记录

1. **增加命名说明和目录结构**：用户要求补充套件和用例的命名规范、目录组织方式
2. **目录改为按功能块组织**：用户建议按功能块组织比套件/用例分离更清晰，已修改为每个功能域一个目录，套件和用例放在一起
3. **变量作用域重新设计**：AI审核发现 M2 与 PRD/M1 存在矛盾（PRD 要求全局变量池，M2 声明用例间不共享）。讨论后决定采用两级作用域：用例级（默认）和跨端口级（export 导出），去掉笼统的"全局变量池"和"用例间变量传递"。已同步修改 M1、M2、PRD 三份文档
