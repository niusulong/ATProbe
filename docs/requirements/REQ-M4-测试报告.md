# M4 测试报告 — 需求文档

> 版本：v0.2（草稿）
> 日期：2026-06-19
> 状态：草稿
> 修订说明：v0.2 配合 M5 成果——§6 颜色开关与压测阈值两项标为已落地（指向 M5）
> 依赖：M3 测试执行引擎（v0.2，§8 执行结果数据结构）

---

## 1. 模块概述

测试报告模块负责消费 M3 执行引擎产出的 `ExecutionResult` 数据，以可读的形式呈现给用户。本模块是执行的"输出端"，本身不参与执行逻辑。

**第一阶段支持的报告格式：**

| 格式 | 用途 | 实时性 |
|------|------|--------|
| **控制台输出** | 执行过程中实时显示进度和结果 | 实时 |
| **HTML 报告** | 执行结束后生成，浏览器查看，可存档 | 事后 |

> JUnit XML（CI/CD 集成）作为预留，不在第一阶段实现。

**核心设计原则：**

- **纯静态 HTML**：无 JS 交互、无外部依赖，浏览器直接打开即可查看。
- **数据驱动**：报告内容完全来自 M3 的 ExecutionResult，不自行产生数据。
- **失败导向**：失败用例的详情和原始日志关联是报告的核心价值（便于排查）。

---

## 2. 数据来源

报告所有内容来自 M3 §8 定义的 `ExecutionResult`：

```
ExecutionResult
├── Summary（概览）
├── CaseResult[]（用例结果）
│   ├── StepResult[]（步骤结果）
│   │   └── AssertionResult[]（断言明细）
│   └── PressureStats（压测统计，若有）
└── LogIndex（原始日志索引）
```

本模块的职责是把这些结构化数据**渲染**为控制台文本和 HTML 页面。

## 3. 控制台输出

### 3.1 两个阶段

控制台输出分两个阶段，对应执行的实时性和事后汇总：

| 阶段 | 时机 | 内容 |
|------|------|------|
| **实时进度** | 执行过程中 | 当前用例/步骤的执行状态，逐行刷新 |
| **结果汇总** | 执行结束后 | 整体统计 + 失败用例列表 |

### 3.2 实时进度格式

执行过程中，订阅 M3 的进度事件（M3 §7.4），逐行输出：

```
[14:30:25] 开始执行: 网络注册测试套件 (共 5 个用例)
[14:30:25] [1/5] AT+CSQ-信号查询
[14:30:25]   → [COM3] AT+CSQ .................. PASS (120ms)
[14:30:26]   → [COM3] AT+CNUM .................. PASS (95ms)
[14:30:26] 用例结果: PASS (耗时 0.3s)
[14:30:26] [2/5] 双模块-TCP通信测试
[14:30:26]   → [COM3] AT+CIPSERVER=1,8080 ...... PASS (110ms)
[14:30:27]   → [COM3] AT+CIFSR ................. PASS (85ms) extract: server_ip=192.168.1.100
[14:30:27]   → [COM5] AT+CIPSTART="TCP",... .... PASS (230ms)
[14:30:27] 用例结果: PASS (耗时 0.5s)
[14:30:28] [3/5] AT+TEST-指令格式测试
[14:30:28]   → [COM3] AT+TEST=1 ................ PASS (90ms)
[14:30:28]   → [COM3] AT+TEST? ................. PASS (88ms)
[14:30:28]   → [COM3] AT+TEST=999 .............. FAIL (95ms) 断言失败: 越界报错
[14:30:28] 用例结果: FAIL (耗时 0.3s)
...
```

**格式要素：**

- 时间戳前缀 `[HH:MM:SS]`。
- 用例进度 `[当前序号/总数] 用例名`。
- 步骤行：`  → [端口] 命令(截断) ...... 结果 (耗时)`，命令过长截断显示。
- 失败步骤：标红（ANSI 颜色码），附简短失败原因。
- extract 的变量值可选显示（`extract: 变量名=值`），便于调试。
- 压测用例：显示进度 `[3/100轮]`，不逐轮打印每步（避免刷屏），定期打印统计快照。

**压测实时输出（特殊处理）：**

```
[14:30:30] [4/5] 压测-AT命令响应时间
[14:30:30]   压测中: 100轮, warmup 5轮
[14:30:35]   进度: 30/100轮 | 成功 30 | 失败 0 | 平均 95ms
[14:30:40]   进度: 65/100轮 | 成功 65 | 失败 0 | 平均 92ms
[14:30:45]   进度: 100/100轮 | 成功 100 | 失败 0
[14:30:45]   统计: 成功率 100% | min 80ms max 210ms avg 95ms P95 130ms P99 180ms
[14:30:45] 用例结果: PASS (耗时 15.2s)
```

### 3.3 结果汇总格式

执行结束后，输出整体汇总：

```
==================== 执行结果汇总 ====================
执行时间: 2026-06-18 14:30:25 ~ 14:31:10 (耗时 45.3s)
用例总数: 5 | 通过 3 | 失败 1 | 跳过 1 | 中断 0
通过率: 75.0%

失败用例:
  [3] AT+TEST-指令格式测试
      步骤3: AT+TEST=999 断言失败: 越界报错 (期望 ERROR, 实际 OK)
      日志: logs/20260618_143025/COM3/case_003.log

跳过用例:
  [5] 网络注册-深度测试 (前置条件失败: AT+CPIN? 未包含 READY)

报告已生成: reports/20260618_143025/report.html
======================================================
```

**汇总要素：**
- 执行时间范围和总耗时。
- 用例统计（总数/通过/失败/跳过/中断）+ 通过率。
- 失败用例列表：用例名 + 失败步骤 + 简短原因 + 原始日志路径。
- 跳过用例列表：用例名 + 跳过原因。
- HTML 报告路径。

### 3.4 控制台输出原则

- **简洁**：实时输出只显示关键信息，详情看 HTML 报告。
- **颜色**：PASS 绿色、FAIL 红色、SKIPPED 黄色、INTERRUPTED 灰色（ANSI 颜色码，可配置关闭）。
- **截断**：命令和响应在控制台截断显示（如命令最多 40 字符），完整内容在 HTML。
- **可关闭颜色**：非终端环境（如重定向到文件）自动关闭颜色，或配置 `--no-color`。

## 4. HTML 报告

### 4.1 报告结构

纯静态 HTML（单文件，内联 CSS，无 JS、无外部依赖），浏览器直接打开。结构分五部分：

```
HTML 报告
├── 1. 报告头（标题、执行时间、整体结果标识）
├── 2. 概览统计（Summary）
├── 3. 用例详情列表（CaseResult[]）
│   ├── 常规用例：步骤表格
│   └── 压测用例：统计表格 + 步骤详情
├── 4. 失败详情（失败用例的详细对比）
└── 5. 原始日志索引（LogIndex，可点击的文件路径）
```

### 4.2 报告头

```
串口AT命令自动化测试报告
执行时间: 2026-06-18 14:30:25 ~ 14:31:10
耗时: 45.3s
整体结果: 部分通过 (通过率 75.0%)
```

整体结果标识：
- 全部通过：绿色 "全部通过"
- 部分通过：黄色 "部分通过"
- 全部失败/跳过：红色 "全部失败"

### 4.3 概览统计

表格展示 M3 Summary 的字段：

| 统计项 | 值 |
|--------|-----|
| 用例总数 | 5 |
| 通过 | 3 |
| 失败 | 1 |
| 跳过 | 1 |
| 中断 | 0 |
| 通过率 | 75.0% |
| 总耗时 | 45.3s |

附加按标签分组的小统计（若有标签）：

| 标签 | 总数 | 通过 | 失败 |
|------|------|------|------|
| network | 2 | 2 | 0 |
| test | 1 | 0 | 1 |
| smoke | 2 | 1 | 1 |

### 4.4 用例详情

每个用例一个区块，按执行顺序排列。用例状态用颜色标识（PASS 绿/FAIL 红/SKIPPED 黄/INTERRUPTED 灰）。

#### 常规用例

```
[FAIL] AT+TEST-指令格式测试                    test/p0 | COM3 | 耗时 0.3s
┌─────┬──────────────────────┬────────┬────────┬─────────────────────────────┐
│ 步骤│ 命令                  │ 端口   │ 结果   │ 详情                        │
├─────┼──────────────────────┼────────┼────────┼─────────────────────────────┤
│ 1   │ AT+TEST=1            │ COM3   │ PASS   │ 包含 OK ✓ (90ms)            │
│ 2   │ AT+TEST?             │ COM3   │ PASS   │ 包含 +TEST: ✓ (88ms)        │
│ 3   │ AT+TEST=999          │ COM3   │ FAIL   │ 越界报错 ✗ (95ms)           │
│     │                      │        │        │   期望: 包含 ERROR           │
│     │                      │        │        │   实际: OK                   │
│     │                      │        │        │   响应: OK\r\n               │
└─────┴──────────────────────┴────────┴────────┴─────────────────────────────┘
原始日志: logs/.../COM3/case_003.log
```

**步骤表格列：**
- 步骤序号
- 命令（完整，不截断）
- 端口
- 结果（PASS/FAIL/SKIPPED，带 ✓/✗ 图标）
- 详情：断言明细（每个断言元素的 name + 期望 + 实际）、响应内容（FAIL 时展示）、耗时、retry 次数（>0 时显示）、poll 次数（>0 时显示）

**展开/收起（纯 CSS 实现）：**
- 步骤的响应内容默认收起（可能很长），用 `<details>` 标签实现点击展开。
- 断言明细默认展示，响应内容收起。

#### 压测用例

```
[PASS] 压测-AT命令响应时间                     stress | COM3 | 耗时 15.2s
压测统计:
┌──────────────┬──────────┐
│ 统计项       │ 值       │
├──────────────┼──────────┤
│ 总轮次       │ 100      │
│ 预热轮次     │ 5        │
│ 计入统计     │ 95       │
│ 成功轮次     │ 95       │
│ 失败轮次     │ 0        │
│ 成功率       │ 100.0%   │
└──────────────┴──────────┘

步骤统计:
┌─────┬──────────┬──────┬──────┬─────┬─────┬─────┬──────┬──────┐
│ 步骤│ 命令     │ 成功 │ 失败 │ min │ max │ avg │ P95  │ P99  │
├─────┼──────────┼──────┼──────┼─────┼─────┼─────┼──────┼──────┤
│ 1   │ AT       │ 95   │ 0    │ 80  │ 210 │ 95  │ 130  │ 180  │
└─────┴──────────┴──────┴──────┴─────┴─────┴─────┴──────┴──────┘
(响应时间单位: ms)
```

**压测步骤统计列**：步骤序号、命令、成功次数、失败次数、min/max/avg/P95/P99（毫秒）。

> 响应时间分布图表（直方图）：纯静态 HTML 无 JS，无法画交互图表。第一阶段用表格展示百分位值；若需直方图，第二阶段考虑引入 Chart.js（需联网或内联库）。

### 4.5 失败详情

失败用例在"用例详情"中已展示失败步骤的对比（期望 vs 实际）。不单独设"失败详情"区，避免重复。失败导向通过以下方式体现：

- 失败用例区块用红色边框/背景突出。
- 失败步骤的断言对比明确展示（期望、实际、响应）。
- 失败用例的原始日志路径直接可点击（相对路径链接）。

### 4.6 原始日志索引

报告末尾展示 LogIndex，按端口分组：

```
原始日志:
  会话目录: logs/20260618_143025/
  COM3:
    case_001.log (AT+CSQ-信号查询)
    case_003.log (AT+TEST-指令格式测试)
  COM5:
    case_002.log (双模块-TCP通信测试)
```

日志路径为相对路径（相对 HTML 报告文件），浏览器中可点击打开（文本文件浏览器能直接显示）。

### 4.7 HTML 报告设计原则

- **单文件**：所有 CSS 内联在 `<style>` 标签，无外部 CSS/JS 文件，便于分发和存档。
- **无 JS**：不依赖 JavaScript。展开/收起用 HTML5 `<details>`/`<summary>` 原生标签。
- **响应式表格**：表格用 CSS，窄屏可横向滚动。
- **颜色语义**：PASS 绿(#4CAF50)、FAIL 红(#F44336)、SKIPPED 黄(#FF9800)、INTERRUPTED 灰(#9E9E9E)。
- **中文支持**：UTF-8 编码，`<meta charset="UTF-8">`。

## 5. 报告管理

### 5.1 报告存储

- 每次执行生成一份 HTML 报告。
- 报告按时间戳命名存储，目录结构：

```
reports/
└── 20260618_143025/              # 执行会话（开始时间）
    └── report.html               # HTML 报告
```

- 报告目录与原始日志目录（`logs/时间戳/`）平级，报告中用相对路径引用日志。

### 5.2 命名规则

- 报告目录名：`YYYYMMDD_HHMMSS`（与日志会话目录一致，便于关联）。
- HTML 文件名固定 `report.html`。

### 5.3 报告保留

- 不自动清理报告文件（与日志保留策略一致，M1 §7.4）。
- 用户手动管理。

### 5.4 历史报告查看（预留）

- PRD 提到"支持历史报告查看和对比（预留）"，第一阶段不实现。
- 后续可通过 M5 CLI 的 `report` 子命令列出/打开历史报告。

## 7. 与其他模块的接口

| 接口 | 方向 | 说明 |
|------|------|------|
| 执行结果 | M3 → M4 | 消费 M3 的 ExecutionResult（M3 §8） |
| 进度事件 | M3 → M4（控制台部分） | 订阅 M3 进度事件做实时输出（M3 §7.4） |
| 原始日志 | M1 → M4（引用） | 报告中引用 M1 记录的原始日志文件路径 |
| CLI | M5 → M4 | M5 调用报告生成、控制台输出开关 |
