# 书写规范（正则 / tags / name 唯一性 / 超时）

写用例时的通用规范，避免踩坑。

## 正则书写规范：单引号 + 单反斜杠

YAML 里正则易因反斜杠转义出错。**统一用单引号字符串**，正则内反斜杠**单写一次**：

```yaml
# 推荐：单引号 + 单反斜杠
extract:
  stat: '\+CEREG:\s*\d,(\d)'

# 不推荐：双引号导致反斜杠需双写
extract:
  stat: "\\+CEREG:\\s*\\d,(\\d)"
```

**原因**：YAML 单引号字符串中反斜杠无特殊含义，原样保留；双引号字符串中反斜杠是转义符，需双写。统一用单引号可显著降低正则出错率。

> 本 skill 的断言核心手段是 `matches` 正则（严格字节级），正则写错会导致断言失效。务必用单引号。

## tags 系统

纯自定义标签，自由定义任意字符串，用于分类筛选和报告分组：

```yaml
tags: [smoke, network, p0]
```

- 执行时通过标签筛选：只执行匹配标签的用例。
- 报告中按标签分组统计。
- 支持多标签组合筛选（交集/并集）。
- 一个用例可有多个标签。

> 本 skill 在 tags 上叠加了命名规范约束：前三段强制为 `[功能块, 指令, 类型]`（详见 SKILL.md），可继续追加 `p0/p1` 等优先级标签。

## name 唯一性作用域

- `name` 在**单个执行范围内**（一次运行加载的所有用例）必须唯一。
- 报告中以"文件名 + name"双标识展示用例，避免不同文件重名混淆。
- 套件靠文件路径引用用例，不依赖 name 唯一性做引用。

## 超时：仅步骤级

- 超时**只在步骤级**配置（步骤的 `timeout` 字段，单位秒）。
- **不存在用例级或全局级超时继承**：步骤未配 `timeout` 时用配置文件的默认值（见 M3 §3）。
- 业务码响应（不以 OK/ERROR 结尾的）**必须设 `timeout`**，否则框架不识别为响应结束，会等满默认超时（详见 SKILL.md「业务码超时陷阱」）。

```yaml
steps:
  - command: 'AT+IPSTATUS=0'
    timeout: 1.2              # 秒，业务码响应必加
    assert: { matches: '^\r\n\+IPSTATUS: 0,DISCONNECT\r\n$' }
```
