# 压测配置（loop 循环）

测指令的响应时间分布、成功率、稳定性时用压测。用例加 `loop` 字段即切换为压测场景（无 `loop` 则是常规执行）。

## 触发规则

用例顶层**存在 `loop` 字段** → 压测执行（steps 作为整体循环执行，统计性能指标）。不存在 → 常规执行。

## loop 配置

```yaml
loop:
  count: 100                   # 循环次数（必填）
  interval: 100                # 循环间隔（毫秒，可选，默认0）
  warmup: 5                    # 预热轮数，默认 0
  abort_on_failure: false      # 遇失败是否中止，默认 false
```

| 字段 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `count` | 数值 | —（必填） | 循环次数 |
| `interval` | 数值(ms) | 0 | 循环间隔 |
| `warmup` | 数值 | 0 | 预热轮数 |
| `abort_on_failure` | 布尔 | false | 遇失败是否中止整个压测 |

## 压测语义（关键，易误解）

| 项目 | 定义 |
|---|---|
| **一轮成功标准（单命令压测）** | 该命令断言通过 = 成功 |
| **一轮成功标准（命令序列压测）** | 一轮内所有步骤断言全通过 = 该轮成功 |
| **压测中失败的处置** | 默认**记一次失败继续**（不中止），最终统计失败率。设 `abort_on_failure: true` 则遇失败即止 |
| **interval 语义** | **"上一轮结束 → 下一轮开始"的间隔**（非固定节拍/非 tick） |
| **warmup 语义** | 前 N 轮**执行但不计入响应时间统计**，避免冷启动污染数据 |

> `interval` 容易误解：它是"轮与轮之间的空隙"，不是"每轮的固定启动时刻"。一轮耗时长则下一轮顺延，不是固定节拍。

## 统计维度

- **单命令压测**：统计该命令的响应时间分布和成功率。
- **命令序列压测**：统计每步的响应时间分布和成功率 + 整体序列成功率。
- **响应时间分布指标**：min / max / avg / **P95 / P99**。

压测场景下内置变量 `{{loop_index}}`（当前循环索引，从 1 开始）可用于 command/data 模板，见 `variables.md`。
