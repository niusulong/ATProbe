# Neoway N58 固件响应规律与严格正则速查

本项目 ch06 实测验证的回码规律（固件 N58-R07-STD-OE_V20_MEITUAN-00F）。
其他指令集（HTTP/FTP/MQTT 等）大概率遵循同样规律，但以实际运行为准。

## 响应骨架（精确到字节）

所有响应以 `\r\n`（0x0D 0x0A）开头，行间用 `\r\n` 分隔。原始字节不经框架规范化（无 strip/replace），
故断言可直接对 `\r\n` 做精确匹配。

### A. 查询/设置成功 —— 以 OK 结尾

格式：`\r\n` + [数据行 + `\r\n`] + `OK\r\n`

| 类型 | 真实响应（可见化：·=空格） | 严格正则 |
|---|---|---|
| 无数据行（设置成功） | `··OK` | `^\r\nOK\r\n$` |
| 数据行1空格 | `··+NETAPN:·"cmnet","",""` | `^\r\n\+NETAPN: "[^"]*","[^"]*","[^"]*"\r\nOK\r\n$` |
| 数据行多空格（右对齐） | `··+XIIC:····0,0.0.0.0`（冒号后4空格） | `^\r\n\+XIIC:    \d+,[\d.]+\r\nOK\r\n$` |
| 取值范围（测试命令） | `··+RECVMODE:·(0-1),(0-1)` | `^\r\n\+RECVMODE: \(0-1\),\(0-1\)\r\nOK\r\n$` |
| 多数值 | `··+TCPKEEPALIVE:·0,120,75,9` | `^\r\n\+TCPKEEPALIVE: \d+,\d+,\d+,\d+\r\nOK\r\n$` |

规律：数据行统一是 `+CMD:` + **恰好1个空格** + `<value>`，XIIC 例外（冒号后4空格，右对齐填充）。
**冒号后的空格数必须实测确认，不能假设。**

### B. 业务码 —— 不以 OK 结尾

格式：`\r\n+CMD:` + 1空格 + `<message>` + `\r\n`（无 OK 尾随）

| 指令 | 真实响应 | 严格正则 |
|---|---|---|
| 链路操作失败 | `··+TCPSETUP:·ERROR` | `^\r\n\+TCPSETUP: ERROR\r\n$` |
| 消息体含空格 | `··+TCPSEND:·SOCKET·ID·OPEN·FAILED` | `^\r\n\+TCPSEND: SOCKET ID OPEN FAILED\r\n$` |
| 状态查询 | `··+IPSTATUS:·0,DISCONNECT` | `^\r\n\+IPSTATUS: 0,DISCONNECT\r\n$` |

> 业务码响应**必须加 `timeout`**（见 SKILL.md 业务码超时陷阱）。

### C. CME 错误码 —— 三类需区分

格式：`\r\n+CME ERROR:` + 1空格 + `<code>` + `\r\n`

| 错误码 | 含义 | 触发场景 |
|---|---|---|
| 53 | 参数无效 / 不允许 | 参数越界、格式错、链路号超范围 |
| 58 | 指令未识别 / 不支持 | 指令名拼错/不完整、固件未编入该功能 |
| 50 | 操作未执行 | 需要 PDP 激活但未激活 |

区分 53 vs 58 很重要：53 = 指令认得但参数错；58 = 指令本身没认出来。

## 严格正则编写要点

1. **`\r\n` 显式写**：在 YAML 正则里写 `\r\n`（反斜杠+r、反斜杠+n），它会被 `re` 引擎解释为
   CR+LF，匹配真实响应里的 0x0D 0x0A。不要写成 `\\r\\n`（双反斜杠会变成字面反斜杠+r）。
2. **空格用字面空格**：写几个空格就匹配几个，**不要用 `\s` 或 `\s*`**（它们会匹配任意数量，
   失去严格性）。在 YAML 单引号字符串里直接敲空格即可。
3. **`^...$` 锚定整段**：`^` 匹配响应开头，`$` 匹配结尾。这保证"恰好"，杜绝多余/缺失字符。
4. **正则元字符转义**：`+` → `\+`，`(` → `\(`，`)` → `\)`，`.` → `\.`（当要匹配字面时），
   `?` 在字符类外要转义。`-` 在字符类内 `[0-9]` 不用转义。
5. **数值占位**：`\d`（1位）、`\d+`（多位）、`[\d.]+`（IP 等）。

## matches vs contains 选择

- `matches`：用正则，做严格格式校验（本 skill 的核心手段）。底层是 `re.search`，但配合
  `^...$` 锚定后等同全匹配。
- `contains`：子串包含，用于"只要出现就行"的宽松场景（如错误码判断 `contains: "+CME ERROR: 53"`）。
- `not_contains`：不包含，用于反证（如确认业务码响应里 `not_contains: "+CME ERROR"`）。

严格字节级校验一律用 `matches` + `^...$`。
